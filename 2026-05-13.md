# 2026-05-13

## 修改记录

- 将 `src/features/movies/components/movies-action-dialog.tsx` 中的“海报地址”输入项改为图片上传流程。
- 新增 `src/api/upload.ts`，封装上传签名获取与阿里云 OSS 直传逻辑。
- 上传成功后自动回填 `posterUrl`，并在电影弹窗中展示海报预览和已保存地址。
- 提交电影创建/编辑表单时继续沿用 `posterUrl` 字段，确保后端接收到上传后的图片地址。
- 更新 `.env.example`，补充 `VITE_UPLOAD_SIGN_URL` 变量说明，便于配置上传签名接口地址。
- 调整 `package.json` 启动脚本，改为通过 `vite --mode` 控制接口请求地址。
- 新增 `.env.local` 和 `.env.production`，分别对应本地与线上接口地址。
- 更新 `.env.example`，补充 `VITE_API_BASE_URL` 配置示例。
- 修正 Vite `mode` 配置，改用默认 `development` 模式，并新增 `.env.development`。
- 删除 `.env.local`，避免其对所有模式生效导致环境地址互相覆盖。
- 修正 OSS 上传签名接口请求方式，改为向 `/movie/upload` 发起 `POST` 请求。
- 参考后端示例调整图片上传字段，OSS 表单签名参数改为小写 `signature`。
- 参考示例简化 OSS 文件名生成规则，使用 `dir + 时间戳 + 扩展名` 作为上传 `key`。
- 增强 OSS 上传失败提示，透传响应体内容，便于定位 bucket、region 或签名配置错误。
- 适配 Vite OSS 代理，开发环境上传改为请求 `/oss`，避免浏览器直连 OSS 时触发跨域限制。

## 说明

- 当前上传签名接口默认使用 `/movie/upload`，也可通过 `VITE_UPLOAD_SIGN_URL` 覆盖。
- 电影弹窗在上传中会禁用提交按钮，避免图片地址尚未回填时提前提交。
- `npm run dev` 默认读取 `.env.development`，`npm run dev:prod` 读取 `.env.production`。
- 本地开发依赖 `vite.config.ts` 中的 `/oss` 代理，修改代理配置后需要重启开发服务。
